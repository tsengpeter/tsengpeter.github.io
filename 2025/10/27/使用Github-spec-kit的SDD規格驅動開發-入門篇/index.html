使用 Github-spec-kit 的 SDD 規格驅動開發 - 入門篇

什麼是 SDD（規格驅動開發）？

SDD（Specification Driven Development，規格驅動開發）是一種軟體開發方法論，其核心理念在於將詳細、可執行的規格文件作為整個開發流程的唯一真實來源（single source of truth）。在這種模式下，我們優先定義「需要構建什麼（what）」以及「為什麼要這樣構建（why）」，然後才交由 AI 編碼助理或開發人員去實現「如何構建（how）」。

SDD 的目標是透過明確、可審查和可演進的技術決策，來減少模糊性、提高代碼品質並加速開發週期。

Github-spec-kit 是什麼？

Github-spec-kit 是由 GitHub 在 2025 年 9 月發布的一個開源工具組，專為輔助開發者與 AI 編碼助理（如 GitHub Copilot, Gemini CLI 等）進行 SDD 開發而設計。它提供了一套結構化的方法與工具，旨在將傳統開發中較為隨性的 “vibe coding”（跟著感覺走）轉變為一個更嚴謹、更可預測的流程。

Github-spec-kit 的核心組件

Github-spec-kit 包含以下幾個關鍵部分：

  1. Specify CLI: 一個命令列介面（CLI）工具，用於管理 SDD 的工作流程、生成專案模板等。
  2. Templates: 提供一組預設的模板，幫助開發者快速創建結構化的規格文件。
  3. Steering Prompts: 一系列精心設計的提示詞（例如 /specify, /plan, /tasks），用於在開發的不同階段引導 AI 編碼助理，使其產出更符合預期的結果。
  4. Constitution.md: 一個專案的「憲法」文件，用於定義專案中不可動搖的原則和指導方針（例如：編碼風格、技術選型、禁止使用的套件等）。AI 在生成代碼時會參考此文件。

使用 Github-spec-kit 的 SDD 工作流程

使用此工具組的開發流程被劃分為四個清晰的、需要逐一驗證的階段：

  1. Specify (規格化): 開發者提供對專案意圖和目的的高層次描述。AI 助理會基於這些描述生成一份詳細的規格文件。
  2. Plan (計畫): 將詳細的規格文件轉化為具體的技術計畫，包含架構選擇、技術棧、數據模型等。
  3. Tasks (任務拆解): AI 助理將技術計畫分解為一系列更小、可執行且可測試的開發任務。
  4. Implement (實現): AI 助理或開發者根據拆解後的任務，逐步完成代碼的編寫與實現。

這種結構化的方法確保了 AI 生成的代碼能夠精確地對應到最初的需求。
